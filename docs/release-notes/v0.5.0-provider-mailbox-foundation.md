# v0.5.0 Provider Mailbox Foundation

## Highlights
- Provider/Mailbox architecture stabilized.
- Multiple Gmail and IMAP mailbox instances supported.
- IMAP provider MVP with persisted config and encrypted credentials.
- Emails can be viewed by mailbox.
- Celery job dispatch model stabilized.
- Sync jobs are mailbox-scoped.
- Digest supports All Mailboxes and Single Mailbox scopes.
- All Mailboxes Digest shows global priority queue and mailbox-grouped summaries.
- Gmail-only Digest issue fixed.

## Celery Reliability
- PostgreSQL is the source of truth for background jobs.
- Redis is used as Celery broker/result backend and mailbox sync lock storage.
- Jobs are committed before Celery dispatch.
- queued means Celery task has been successfully dispatched.
- celery_task_id is recorded.
- dispatch_failed state handles send_task failures.
- Orphan/stale/completed Celery tasks are safely ignored.
- UnpickleableExceptionWrapper removed from sync/digest business-error path.

## Digest Scope
- scope_type=all supports all connected mailboxes.
- scope_type=mailbox supports a specific mailbox.
- Digest selector uses GET /api/mailboxes.
- All Mailboxes is the default selector option.
- Digest items include mailbox source metadata.

## Manual Smoke Passed
- 2 IMAP + 1 Gmail sync passed.
- Stop worker → queued jobs → restart worker → jobs drained passed.
- All Mailboxes Digest passed.
- Gmail mailbox Digest passed.
- IMAP mailbox Digest passed.

## Known Limitations
- Real Outlook provider remains out of scope.
- Cross-mailbox thread merging is not implemented.
- Mail sending is not implemented.
- AI Settings UI is not implemented.
- Celery Beat is not implemented.
- IMAP edit/update-password advanced flows may remain future work if not already completed.
