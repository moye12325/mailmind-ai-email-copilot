# MailMind v0.3.0 Async Redesign

- Release name: MailMind v0.3.0 Async Redesign
- Base version: v0.2.0-digest-ai
- Merged QA branch: integration/v0.3-async-redesign-qa
- Manual QA: passed, confirmed by user

## Backend Changes

- Background jobs foundation: Celery worker entrypoint (`app.jobs.worker:app`), Redis broker/result backend.
- Job Status API: `GET /api/jobs`, `GET /api/jobs/{job_id}`, `POST /api/jobs/{job_id}/retry`.
- Async mail sync job: `POST /api/mailboxes/{mailbox_id}/sync-jobs` enqueues an `email_sync` job.
- Async digest generate job: `POST /api/digest/today/generate-jobs` enqueues a `digest_generate` job.
- Async digest refresh job: `POST /api/digest/today/refresh-jobs` enqueues a `digest_refresh` job.
- Job retry / failure handling: `POST /api/jobs/{job_id}/retry` with `max_retries = 3`, `409 JOB_RETRY_LIMIT_EXCEEDED` when limit reached.
- Scheduled email sync / scheduled digest local MVP foundation: `app.jobs.scheduled_email_sync` and `app.jobs.scheduled_digest` tasks.
- Error redaction for job failures: `error_message` is sanitized before storage and API response; no tokens, keys, or raw prompts leak.
- Public job type normalization: internal types (`sync_today_emails`, `generate_daily_digest`, etc.) are mapped to public values (`email_sync`, `digest_generate`, etc.).
- Public job status normalization: internal `succeeded` is mapped to `completed`.

## Frontend Changes

- Disabled / clickable state fix: buttons and interactive elements correctly reflect disabled states.
- Avatar account menu: new `<AccountMenu>` component with user avatar, dropdown, and sign-out.
- Amber / Ink visual identity: redesigned color palette and visual language.
- Theme system redesign: four themes — Amber Focus, Noir Pulse, Paper Calm, Dense Minimal.
- i18n foundation: `<LanguageProvider>`, `useLanguage()` hook, locale JSON files.
- English / Chinese language resources: `en.json` and `zh.json` with full UI string coverage.
- UI consistency pass: aligned component styles, spacing, and interaction patterns.
- Runtime UI regression fixes: nullable email data handling, theme overflow fix.
- New contract tests: `account-menu.contract.test.tsx`, `digest-dashboard.contract.test.tsx`, `button.contract.test.tsx`, `i18n.contract.test.ts`.

## API Changes

New routes (additive to v0.2):

- `POST /api/mailboxes/{mailbox_id}/sync-jobs` — async email sync job.
- `POST /api/digest/today/generate-jobs` — async digest generate job.
- `POST /api/digest/today/refresh-jobs` — async digest refresh job.
- `GET /api/jobs` — list current user's jobs with filters (`limit`, `offset`, `job_type`, `status`, `created_from`, `created_to`).
- `GET /api/jobs/{job_id}` — get single job detail.
- `POST /api/jobs/{job_id}/retry` — retry a failed job.

Existing v0.2 synchronous routes remain unchanged:

- `POST /api/mailboxes/{mailbox_id}/sync` (synchronous).
- `POST /api/digest/today/generate` (synchronous).
- `POST /api/digest/today/refresh` (synchronous).

## Database / Migration Changes

- No new tables or migrations added in v0.3.
- Alembic head remains `20260620_0007`.
- v0.3 reuses the existing `sync_jobs` table with additional columns for `retry_count`, `error_code`, `error_message`, `payload_json`, `trigger_source`, `started_at`, `finished_at`.
- These columns were already present in the v0.2 schema.

## Worker / Runtime Changes

- Celery worker: `uv run celery -A app.jobs.worker:app worker --loglevel=info --pool=solo`.
- Redis is the default broker and result backend via `REDIS_URL`.
- `BACKGROUND_JOBS_EAGER=true` runs tasks in-process for tests.
- Scheduled tasks: `celery call app.jobs.scheduled_email_sync` and `celery call app.jobs.scheduled_digest`.
- Celery Beat is not implemented; scheduled tasks require manual or external trigger.

## Security Notes

- Job `error_message` is redacted before storage and API response.
- Job responses do not contain access tokens, refresh tokens, API keys, cookies, authorization headers, raw email bodies, raw Gmail payloads, raw prompts, or raw model responses.
- `retry_count` and `max_retries` are exposed; retry logic enforces ownership checks.

## Known Limitations

- Scheduled jobs are local MVP foundation and require external/manual trigger. Celery Beat is not implemented.
- Production-grade distributed scheduling is not included.
- AI Provider settings are still env-based; no full user-managed settings UI.
- Outlook / IMAP are not included.
- Multi-mailbox aggregate digest is not complete.
- Production deployment, Google OAuth verification, and security review are not complete.
