# MailMind v0.2.0 Digest AI Release Notes

Release tag: `v0.2.0-digest-ai`

This release promotes the v0.2 Digest AI integration branch. It connects the Daily Digest frontend to the backend, adds real AI provider support behind the existing provider abstraction, improves Gmail/email query behavior, and keeps the project in local validation scope rather than production SaaS scope.

## Implemented

- Dashboard Daily Digest UI wired to `GET /api/digest/today`.
- Frontend digest generate and refresh actions.
- Digest summary and digest item rendering.
- Digest item `mark-done`, `dismiss`, and `snooze` actions.
- `/actions` action history page with filters and pagination.
- Email list search, read-state filters, date filters, mailbox filters, and pagination.
- Email detail UX polish and read/unread feedback improvements.
- Mailbox settings state polish.
- Environment-configured OpenAI-compatible AI provider profiles.
- AI provider fallback ordering through environment variables.
- AI provider/model metadata recorded on `ai_runs`.
- Stronger digest prompt contract for real provider output.
- More tolerant digest parser aliases for common real-provider field names.
- Safe digest failure diagnostics while keeping public API errors generic.
- Gmail sync error mapping improvements.
- Secret redaction helpers and redaction coverage for AI/provider failures.
- v0.2 contract examples for email, actions, and AI provider configuration.

## API Surface

The v0.2 backend includes:

- Auth routes under `/api/auth`.
- Gmail OAuth routes under `/api/auth/gmail`.
- Mailbox routes under `/api/mailboxes`.
- Email routes under `/api/emails`, including query filters on `GET /api/emails`.
- Digest routes under singular `/api/digest`.
- Digest item action routes under `/api/digest/items/{item_id}`.
- User action routes under `/api/actions`, including filtering and pagination.
- Health check at `/health`.

See `docs/api/CURRENT_API_SUMMARY.md`.

## Configuration

Real AI provider calls are configured through local environment variables only:

- `AI_PROVIDER_MODE`
- `AI_DEFAULT_PROVIDER`
- `AI_PROVIDER_ORDER`
- `AI_PROVIDER_<ID>_TYPE`
- `AI_PROVIDER_<ID>_BASE_URL`
- `AI_PROVIDER_<ID>_API_KEY`
- `AI_PROVIDER_<ID>_MODEL`
- `AI_PROVIDER_<ID>_TIMEOUT_SECONDS`
- `AI_PROVIDER_<ID>_MAX_RETRIES`

Do not commit real provider keys, Google secrets, Gmail tokens, or `APP_ENCRYPTION_KEY`.

## Known Limitations

- Real AI provider calls depend on local network and provider availability.
- AI provider configuration is environment-only; there is no in-app provider settings UI.
- Digest generation is synchronous.
- Celery and scheduled sync are not implemented.
- Multi-mailbox aggregate digest is not complete.
- Outlook and IMAP are not implemented.
- Production deployment, Google OAuth verification, and security review are not complete.
